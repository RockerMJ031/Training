Claude 为什么烧 Token

3 分钟搞懂 · Purple Ruler 团队内部培训

核心事实：Claude 没有记忆

Claude 不是一个"一直在那里"的人。每次你发新消息，它会从头重读整个对话。

第 1 轮  你发 100 字 → Claude 读 100 字 → 回复 200 字
第 2 轮  你发 50 字  → Claude 读 350 字（100+200+50）→ 回复 200 字
第 3 轮  你发 30 字  → Claude 读 580 字（全部历史）→ 回复 200 字
...
第 20 轮 你发 30 字 → Claude 读 5,800 字（全部历史）→ 回复 200 字

🧠 类比：失忆的天才顾问

Claude 就像一个极聪明但每隔 5 秒就失忆的顾问。每次你问他问题，助理必须把之前所有对话记录从头念给他听，他才能回答。对话越长，念的越多，花的越多。

同样 20 轮对话，拆开聊 vs 一路聊

❌ 1 个长对话 · 20 轮
63,000
tokens 总消耗
✅ 4 个短对话 · 各 5 轮
18,000
tokens 总消耗 — 省 3.5 倍

对话越长，每条消息越贵。总成本按轮数的平方增长。

粘贴代码 vs 放进 Project：天壤之别

很多人把整个文件的代码直接粘到聊天框里。这和把文件放进 Project 的成本差距巨大：

❌ 粘贴到聊天框（1000 行代码）
第 1 轮： Claude 读 1000 行 + 你的问题
第 2 轮： Claude 读 1000 行 + 之前的问答 + 你的新问题
第 3 轮： Claude 读 1000 行 + 所有历史 + 新问题
...
→ 那 1000 行代码被重读了 N 遍，每轮都收费

✅ 放进 Project Knowledge
第 1 轮： Claude 读 1000 行（缓存生效）+ 你的问题
第 2 轮： 1000 行已缓存，不重复计费 + 新问题
第 3 轮： 1000 行已缓存，不重复计费 + 新问题
...
→ 文件只收一次费，后面每轮只算你的新问题
❌ 粘在聊天里 · 聊 10 轮
10x
那份代码被读了 10 遍，每遍都收费
✅ 放 Project Knowledge · 聊 10 轮
1x
缓存生效后只算一次，后续免费

📚 类比：复印 vs 原件

粘在聊天里 = 每次开会都把整份文件重新复印一遍给所有人。
放进 Project = 会议室里有一本永久参考手册，所有人随时翻，不用重印。

结论：任何你需要 Claude 反复参考的内容（公司文档、代码文件、SOP、模板）→ 放 Project Knowledge，不要粘到聊天里。

6 个省 Token 的习惯

1

做完一件事就开新对话

❌ 一个对话从周一用到周五

✅ 每个任务一个对话，做完就开新的

2

只粘相关段落，不粘整篇文档

❌ 把 5 页邮件全部粘进去

✅ 只粘你需要 Claude 处理的那一段

3

第一条消息就说清格式要求

❌ "写个总结" → "改表格" → "加标题" → "换英文" → "还是中文"

✅ "写个中文总结，表格形式，200 字以内，重点加粗"

4

问题要具体，避免来回澄清

❌ "帮我写个东西" → "什么东西？" → "培训的" → "哪个培训？"

✅ "帮我写上周三 ESL 培训的会议总结，重点是教材反馈，200 字"

5

先要大纲，别直接要长文

❌ "写一份完整培训手册" → 输出 3000 字，你只用了 2 段

✅ "先给 5 个要点的大纲" → 确认方向 → 再展开

6

用 Projects 存公司背景，不用每次重说

❌ 每次新对话："我是 Purple Ruler 的，我们做 SEND tutoring…"

✅ 左侧创建 Project → 把背景写进 Instructions → 以后自动带

一句话原则：

把 Claude 当按字收费的顾问
给的背景越精准、要求越具体、对话越短
花的越少
